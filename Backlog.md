Projeto PetLife - Backlog Geral
1. Prioridade Alta (Responsável: Camargo)
Tarefas críticas de fundação, arquitetura e segurança.

Definição do escopo: Documentação dos objetivos do PetLife e o que o sistema se propõe a resolver.

Diagrama de Casos de Uso: Mapeamento visual de como o Atendente e o Admin interagem com o sistema.

Modelagem do Banco de Dados: Desenho das tabelas (Clientes, Pets, Agendamentos) e suas relações.

Requisitos Funcionais: Listagem técnica de todas as funções que o sistema deve executar obrigatoriamente.

Requisitos Não-Funcionais: Definição de metas de performance, segurança de dados e tempo de resposta.

Interface de Cadastro de Clientes: Planejamento dos campos de texto, validação de CPF e máscaras de telefone.

Interface de Cadastro de Pets: Definição de formulários para Raça, Idade e histórico de comportamento.

Regras de Agendamento: Criação da lógica para evitar choque de horários entre diferentes profissionais.

Fluxo de Login: Planejamento da tela de entrada e níveis de permissão para cada funcionário.

Recuperação de Senha: Definição do processo de envio de e-mail seguro para troca de senha esquecida.

2. Prioridade Média (Responsável: Sazon)
Funcionalidades operacionais e de interface com o usuário.

Módulo de Vendas: Planejamento da interface de PDV (frente de caixa) para venda de rações e acessórios.

Controle de Estoque: Lógica de contagem de produtos e aviso automático de itens em falta.

Lista de Serviços: Organização do catálogo de preços para banho, tosa e consultas veterinárias.

Dashboard Principal: Planejamento visual dos cards que mostram os agendamentos do dia.

Sistema de Alertas: Lógica para destacar pets que possuem vacinas ou vermífugos vencidos.

Histórico de Saúde do Pet: Criação de um campo de "prontuário" para registrar alergias ou observações.

Busca e Filtros: Planejamento da ferramenta de pesquisa rápida de clientes por nome ou código.

Permissões de Acesso: Definição do que o Tosador pode ver em comparação ao Gerente da loja.

Comprovante de Agendamento: Modelo de ticket/recibo para o cliente levar após marcar um serviço.

Integração de Pagamentos: Planejamento do fluxo de fechamento de conta com PIX e Cartões.

3. Prioridade Baixa (Responsável: NegoLoko)
Relatórios, otimizações e suporte.

Relatório de Faturamento: Planejamento de gráficos mensais de ganhos e perdas.

Relatório de Mais Vendidos: Levantamento de dados para saber quais serviços/produtos saem mais.

Sistema de Fidelidade: Lógica de pontuação onde o cliente ganha brindes após X serviços realizados.

Notificações via WhatsApp: Planejamento de mensagens automáticas para lembrar o cliente do banho.

Cadastro de Fornecedores: Tela para gerenciar contatos de onde a loja compra as rações.

Política de Cancelamento: Definição das regras e prazos para o cliente desmarcar um horário.

Backup de Dados: Planejamento de cópias de segurança automáticas para evitar perda de cadastros.

Estudo de Responsividade: Plano para garantir que o sistema funcione bem em tablets e celulares.

FAQ/Ajuda: Criação de um guia de dúvidas frequentes para o uso do sistema pelos funcionários.

Fechamento de Caixa: Lógica para conferência de valores no final do expediente diário.
